# План семинара 2

## Название

**Семинар 2. От чата к простому агенту: функции, инструменты и цикл действий**

## Длительность

80 минут.

## Цель семинара

К концу семинара школьник должен понять:

- что Python-функция может быть инструментом;
- что модель может не только отвечать, но и выбирать действие;
- что инструмент можно описать текстом;
- что модель может вернуть JSON-команду;
- что программа может прочитать эту команду и вызвать нужную функцию;
- что простой агент — это цикл: вопрос → решение → инструмент → наблюдение → финальный ответ.

Главный результат:

> Ученик собирает простого агента, который выбирает один из нескольких инструментов и использует результат инструмента в ответе.

## Предпосылки

Семинар 2 опирается на семинар 1.

Ученик уже видел:

- `messages`;
- запрос к модели;
- функцию `ask_model`;
- JSON-ответ;
- историю сообщений.

## Базовая траектория

```text
чат → Python-функция → инструмент → JSON-команда → вызов инструмента → простой агент
```

## Пример инструментов

Для первого варианта достаточно 3 инструментов:

```python
def add(a, b):
    return a + b

def multiply(a, b):
    return a * b

def explain_term(term):
    dictionary = {
        "API": "Способ, с помощью которого одна программа обращается к другой.",
        "JSON": "Формат для передачи структурированных данных.",
        "агент": "Программа, где модель может выбирать действия."
    }
    return dictionary.get(term, "Такого термина пока нет в словаре.")
```

## Идея tool registry

Инструменты удобно хранить в словаре:

```python
tools = {
    "add": add,
    "multiply": multiply,
    "explain_term": explain_term,
}
```

Тогда программа может вызвать функцию по имени:

```python
tool_name = "multiply"
result = tools[tool_name](3, 7)
```

## Упрощённый tool calling

Не обязательно использовать настоящий function calling API.

Для учебного семинара можно попросить модель вернуть JSON:

```json
{
  "tool": "multiply",
  "arguments": {
    "a": 3,
    "b": 7
  }
}
```

Потом Python:

1. читает JSON;
2. достаёт `tool`;
3. достаёт `arguments`;
4. вызывает нужную функцию;
5. передаёт результат обратно модели.

## Главная схема агента

```text
Пользователь задаёт вопрос
↓
Модель решает, нужен ли инструмент
↓
Python вызывает инструмент
↓
Результат инструмента добавляется в историю
↓
Модель даёт финальный ответ
```

## Что важно не усложнять

Во втором семинаре не нужно начинать с LangChain или LangGraph.

Сначала нужно показать саму механику:

> Агент — это не магия.  
> Это программа, где модель может выбирать следующий шаг, а Python выполняет этот шаг.
