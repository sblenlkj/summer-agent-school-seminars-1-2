## Длительность

80 минут.

## Цель семинара

К концу семинара школьник должен понять:

- что модель можно вызывать из программы;
- что запрос к модели — это HTTP-запрос;
- что GigaChat использует схему `AUTH_KEY` → `ACCESS_TOKEN` → запрос к модели;
- что промпт влияет на ответ;
- что ответ модели приходит как JSON;
- что чат — это список сообщений;
- что историю сообщений можно хранить в Python и передавать модели.

Главный результат:

> Ученик сам запускает запрос к модели и собирает простой мини-чат в Colab.

## Методическая логика

Не нужно пытаться рассказать всё про API, OAuth и безопасность.

На семинаре важно показать простую цепочку:

```text
текст пользователя → Python → GigaChat API → ответ модели → Python
```

Потом расширить её до истории:

```text
список сообщений → GigaChat API → новый ответ → добавляем ответ в список
```

## Ключевые объяснения

### API

API — это способ, с помощью которого одна программа обращается к другой.

В нашем случае:

- Colab запускает Python-код;
- Python отправляет запрос через интернет;
- сервер GigaChat возвращает ответ;
- Python достаёт из ответа нужный текст.

### GigaChat-токены

Важно развести два понятия:

- `AUTH_KEY` — ключ авторизации из личного кабинета;
- `ACCESS_TOKEN` — временный токен, который мы получаем через API и используем в запросах к модели.

Для школьников достаточно аналогии:

> `AUTH_KEY` — долгосрочный пропуск.  
> `ACCESS_TOKEN` — временный билет на ближайшие запросы.

### Чат

Чат — это не магия.

В простейшем виде чат — это список сообщений:

```python
messages = [
    {"role": "user", "content": "Привет!"},
    {"role": "assistant", "content": "Привет! Чем помочь?"}
]
```

Чтобы модель учитывала контекст, мы каждый раз отправляем ей весь список `messages`.
